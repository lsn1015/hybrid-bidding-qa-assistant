## 2. 产品概述

### 2.1 产品背景

招投标行业特点：

* 信息分散，企业需跨平台检索
* 招标文件内容重复，但关键要求分散，人工核对成本高
* 投标流程依赖经验，缺乏系统化支持
* 企业缺乏数据分析能力，难以评估投标策略

**解决方案**：构建多模型协作的智能问答助手

* **LLM**：中枢模型，负责意图识别、智能回复
* **Embedding 模型**：向量化处理
* **Reranker 模型**：结果验证
* **价值**：提高信息获取效率，降低人工成本，辅助投标决策

### 2.2 产品定位

* **目标用户**：企业、招标代理机构
* **核心能力**：

  * 信息检索（找标）
  * 文件理解（读标）
  * 智能问答（规则咨询）
  * 决策支持（投标建议）

### 2.3 AI产品愿景

构建企业级私有知识驱动的智能招投标系统，实现：

* 基于企业私有数据的精准问答
* 招标项目智能匹配
* 投标流程智能化辅助
* 企业投标能力数据化提升

### 2.4 核心价值主张

* 投标准备时间 ↓ ~50%（估算）
* 投标决策效率 ↑ ~30%
* 问答响应时间 < 1s
* 问答相关性 > 90%

## 4. 产品功能设计

### 4.1 核心功能模块

| 功能          | 描述                      | 实现方案                       |
| ----------- | ----------------------- | -------------------------- |
| 招标/中标公告智能查询 | 用户自然语言查询招标/中标信息，返回结构化结果 | RAG + 向量检索 + MySQL + LLM校验 |
| 供应关系查询      | 查询企业历史投标关系与供应链结构        | 知识图谱+ NL2SQL + MySQL   |
| 投标建议生成      | 基于招标文件和历史数据生成投标建议       | RAG + 外部数据接口（企查查等） + LLM推理 |
| 法律法规检索      | 查询政府采购相关法规并生成解释         | 法规知识库 + RAG                |

* **目的**：解耦 LLM 与数据库，提高系统可控性，降低错误风险

```json
    "phone": "138xxxx"
  }
}
```

## 10. 竞品分析

### 10.1 信息平台类

* 中国政府采购网：信息权威，无AI分析

### 10.2 数据工具类

* 企查查、天眼查：企业关系强，但无投标智能问答

### 10.3 通用AI模型

* ChatGPT、通义千问：自然语言强，但行业数据接入不足
